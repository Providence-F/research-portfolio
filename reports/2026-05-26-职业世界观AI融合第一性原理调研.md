职业世界观 × AI 融合：第一性原理深度调研

> **背景**：为2026年栖霞青年大学生创新创业大赛「数字赋能赛道」做准备
> **约束**：5.31报名截止 + 6.15网站上线 + 一人执行 + 纯前端无后端
> **目标**：找到AI与认知框架的真实结合点，而非"AI套壳"

## 步骤1：拆解到不可再分的最小认知单元

### 1.1 大学生就业迷茫的本质

**一句话本质**：需要做出一系列半不可逆决策（选行业/选职能/选城市/选路径），但缺乏做出这些决策的认知基础。

迷茫不是"没有信息"，而是：
- 不知道自己要什么（价值观模糊）
- 不知道自己能做什么（能力边界不清）
- 不知道选项长什么样（信息二手且碎片化）
- 不知道怎么验证判断（没有低成本试错机制）

### 1.2 用户需要做的5个核心决策

| 决策 | 决策层级 | 问题本质 |
|------|----------|----------|
| 选择赛道/方向 | 战略层 | 行业×职能的组合空间，哪个对我开放？ |
| 评估自己 | 战略层 | 我的能力、兴趣、价值观分别是什么？ |
| 验证假设 | 战术层 | 我的判断对不对？怎么低成本试？ |
| 制定路径 | 战术层 | 从这里到那里，怎么走？ |
| 迈出第一步 | 执行层 | 明天早上我具体做什么？ |

### 1.3 最小认知单元清单（8个）

| # | 认知单元 | 决策层级 | 当前用户如何完成 | "不可分"性 |
|---|----------|----------|------------------|------------|
| 1 | **价值观排序** | 战略 | 自己想/被社交媒体影响/从不认真想过 | 最终输出是主观偏好，无法客观测量——必须用户自己判断"什么对我重要" |
| 2 | **能力边界感知** | 战略 | 凭感觉/问同学/偶尔被老师评价 | 能力的"感知"vs"真实水平"是两件事，但用户只能感知到感知——这是起点 |
| 3 | **能量识别** | 战略 | 自己隐约知道但说不清楚 | 做什么不累/做什么消耗，这是主观体验，无法外包 |
| 4 | **选项分布认知** | 战略 | 小红书/学长学姐/招聘信息 | 知道"有什么工作"是信息问题，可被结构化——但必须是真实信息而非二手加工 |
| 5 | **概率校准** | 战术 | 凭感觉/看别人经验 | "我拿到这个offer的概率是多少"——需要结合个人条件+市场数据，用户自己算不准 |
| 6 | **风险偏好判断** | 战术 | 不知道自己能承受多少风险 | 稳定vs高薪vs兴趣的三角取舍——必须知道自己的底线 |
| 7 | **路径规划** | 战术 | 搜攻略/问前辈 | "从A到B怎么走"——需要拆解成可执行的步骤 |
| 8 | **最小行动识别** | 执行 | 想太多/卡在"不知道从哪开始" | "明天早上做什么"——必须具体到可以立刻开始 |

### 1.4 关键洞察

**这8个单元中，有4个是"AI完全无法介入的主观判断"**（#1价值观排序、#2能力边界感知、#3能量识别、#6风险偏好判断）。

AI能介入的是另外4个：
- #4 选项分布认知 → 信息结构化
- #5 概率校准 → 数据+推理
- #7 路径规划 → 框架+案例
- #8 最小行动识别 → 拆解+适配

**但问题是**：用户卡住的地方往往是那4个主观判断，而非信息不足。

**这就是为什么"推荐系统"式的产品对这个场景无效——用户需要的不是"答案"，而是"想清楚的工具"。**

# 步骤2：每个认知单元，AI到底能做什么

| 认知单元 | AI能不能介入？ | 介入方式（具体机制） | 介入后比现在好多少？ | AI能做到但人做不了的？ | 人必须做的（AI替代不了的） | 实现所需（我的约束下） |
|---------|--------------|---------------------|---------------------|-----------------------|--------------------------|----------------------|
| **#1 价值观排序** | 有限介入 | AI通过苏格拉底式追问帮用户澄清模糊的价值观。不是"给答案"，而是"帮你看清自己已经知道的东西"。具体：追问"你说想要自由，那如果收入减半你还愿意吗？" | 比用户自己想清晰30-50%——因为人很难客观审视自己的假设 | AI可以连续追问而不疲劳、不带偏见、不因关系而不敢说 | 最终排序必须用户自己做 | Prompt工程 + LLM调用（纯前端可实现） |
| **#2 能力边界感知** | 有限介入 | AI通过分析用户描述的经历，提取能力信号。比如用户说"我带着小组做了个XX项目"，AI可以识别出"项目管理、跨部门协调、进度控制"等能力标签。 | 比用户自己描述更结构化，但准确性取决于用户描述质量 | AI可以把模糊经历翻译成能力语言（JD语言） | 用户必须提供真实经历，不能编造 | Prompt + LLM |
| **#3 能量识别** | 有限介入 | AI通过"什么时候不累？什么时候想逃避？"这类问题，帮用户识别模式。关键是问对问题。 | 比用户自己反思更系统，但能量感是主观的 | AI可以跨多个维度做模式识别 | 用户必须诚实回答 | Prompt + LLM |
| **#4 选项分布认知** | 强介入 | **这是AI最适合的领域**。AI可以：①将行业×职能×城市拆解成结构化数据；②根据用户条件筛选可行选项；③用真实案例说明每个选项的"真实一天"。 | 比小红书碎片信息好10倍——因为是结构化的、可筛选的、真实的 | AI可以实时更新选项库（人工做不到） | 用户需要验证AI给出的信息是否真实 | **需要数据源**——纯前端方案可以用预置数据+LLM推理 |
| **#5 概率校准** | 强介入 | AI可以：①基于用户条件（学校/专业/经历）估算不同方向的匹配度；②用类似背景的成功案例做参照；③给出"保底/冲刺/理想"三档预估。 | 比用户自己估准确50%+——因为人有严重的乐观/悲观偏差 | AI可以跨案例做统计推理 | 用户需要决定"接受哪个概率水平" | Prompt + LLM + 案例库 |
| **#6 风险偏好判断** | 有限介入 | AI通过情景测试帮用户校准："如果月薪减半但做喜欢的事，你能接受多久？""如果3个月没收入，你会怎么办？" | 比用户自己想更具体，但最终是主观判断 | AI可以设置极端情景迫使用户表态 | 用户必须对自己诚实 | Prompt + LLM |
| **#7 路径规划** | 中等介入 | AI可以：①将大目标拆成小步骤；②基于案例给出具体时间线；③识别关键节点和风险点。 | 比搜攻略更个性化，因为是基于用户具体条件的 | AI可以同时考虑多个变量（时间/金钱/风险/兴趣）做路径优化 | 用户必须评估"这条路我真的想走吗" | Prompt + LLM + 路径模板库 |
| **#8 最小行动识别** | 强介入 | AI可以：①把"找实习"拆成"明天上午做这1件事"；②基于用户当前位置/能力给出可执行的第一步；③用15分钟可完成为标准筛选行动。 | 比"你去试试看"具体100倍——用户知道明天早上醒来做什么 | AI可以把抽象目标翻译成具体行动 | 用户必须执行——AI不能替他做 | Prompt + LLM + 行动库 |

## 关键结论

**AI在这个场景下的真实价值排序**：

1. **#4 选项分布认知** → AI最大价值点（信息结构化 + 实时更新）
2. **#5 概率校准** → AI第二大价值点（跨案例推理）
3. **#8 最小行动识别** → AI第三大价值点（具体化 + 可执行）
4. **#1 价值观排序** → AI辅助价值（苏格拉底追问）
5. **#2 能力边界感知** → AI辅助价值（经历→能力翻译）
6. **#3 能量识别** → AI辅助价值（模式识别）
7. **#6 风险偏好判断** → AI辅助价值（情景测试）
8. **#7 路径规划** → AI中等价值（拆解+个性化）

**诚实判断**：AI在这个场景下不是"万能的"，而是在**信息处理+推理+具体化**三个环节有真实价值。在**价值观判断+能力感知+能量识别**等主观环节，AI只能做"追问者"而非"回答者"。

## 步骤3：从AI介入形态倒推产品形态

### Q1：AI主导整个流程，产品应该长什么样？

#### 三种交互范式对比

| 范式 | 适合的认知单元 | 优势 | 劣势 | 我的技术能力能否实现 |
|------|---------------|------|------|---------------------|
| **A. 对话式引导** | #1价值观排序、#2能力边界、#3能量识别、#6风险偏好 | 深度探索、个性化、用户参与感强 | 摩擦大、完成率低、用户可能中途放弃、Coze v1已验证"追问疲劳" | ✅ Coze Bot已验证；纯前端可调用LLM API |
| **B. 卡片式自助** | #4选项分布、#7路径规划、#8最小行动 | 低摩擦、可浏览、完成率高、视觉友好 | 缺乏深度、用户可能"看了但没想"、当前v1/v2形态 | ✅ 这是v1/v2已有的形态 |
| **C. 混合式** | 所有单元 | 兼顾深度和低摩擦、可分阶段进行 | 架构复杂、状态管理难度高 | ⚠️ 需要前端状态管理，但可行 |

#### 推荐方案：C（混合式）

**理由**：

1. **用户旅程是分阶段的**：不是所有认知单元同时激活
   - 第一阶段（5分钟）：卡片式自助 → 快速了解选项分布、完成简单自测
   - 第二阶段（15-30分钟）：对话式引导 → 深度探索价值观、能力、能量
   - 第三阶段（5分钟）：卡片式输出 → 生成个性化报告、推荐路径、具体行动

2. **符合你的认知框架**：
   - 道法层（价值观/能力/能量）→ 对话式引导
   - 术器层（选项/路径/行动）→ 卡片式自助

3. **可实现性**：
   - 前端组件复用Lovable生成的卡片组件
   - 对话部分可以用Coze Bot嵌入，或纯前端调用LLM API
   - 状态管理用LocalStorage/IndexedDB存储用户数据

#### 具体产品形态

```
用户入口（v2网站）
    ↓
【第一阶段：快速自测】卡片式，5分钟
  - 用户填5-8个核心问题（专业/兴趣/焦虑来源）
  - 前端本地处理，无需调用API
  - 输出：初步用户画像（粗糙但够用）
    ↓
【第二阶段：AI引导探索】对话式，15-30分钟
  - AI基于第一阶段数据，主动追问
  - 按"价值观→能力→能量→风险"顺序探索
  - 输出：完整用户画像
    ↓
【第三阶段：个性化输出】卡片式，5分钟
  - AI生成：3个推荐方向 + 每个方向的匹配理由 + 路径拆解 + 第一步行动
  - 用户可保存/下载报告
    ↓
【后续：行动追踪】（可选）
  - 用户记录行动进展
  - AI定期提醒/校准
```

### Q2："数字赋能赛道"的竞赛定位

#### 评审视角的5个加分项

1. **AI是核心驱动力，不是装饰**
   - ✅ 你的项目：AI做引导推理，不是"加个AI聊天框"
   - ❌ 扣分项：只是在现有内容上加了个AI问答功能

2. **有真实用户价值，不是炫技**
   - ✅ 你的项目：解决大学生真实痛点，有用户调研数据支撑
   - ❌ 扣分项：技术很酷但不知道给谁用

3. **有落地可行性，不是PPT**
   - ✅ 你的项目：v1/v2已有代码，有Coze Bot验证，纯前端可部署
   - ❌ 扣分项：需要大量后端/数据/团队才能实现

4. **有差异化，不是跟风**
   - ✅ 你的项目：认知层蓝海，引导式对话+个人画像没有产品做对
   - ❌ 扣分项：又一个职业测评/又一个AI简历工具

5. **社会价值可量化**
   - ✅ 你的项目：帮助大学生减少就业焦虑、提高决策质量
   - ❌ 扣分项：不知道帮了谁、帮了多少

#### 3个扣分点（需要规避）

1. **"AI幻觉"风险**：推荐错误的职业方向 → 需要兜底机制（见步骤5）
2. **"伪AI"嫌疑**：评委可能觉得"这不就是个问卷+规则引擎吗" → 需要展示真实的AI推理过程
3. **"数据孤岛"问题**：没有用户数据验证效果 → 需要在demo中展示用户反馈（即使是小样本）

## 步骤5：风险评估与边界

### 5.1 AI层面的风险

#### 风险1：AI诊断的准确性如何验证？用户信不信？

**风险等级**：中等

**现状**：
- AI诊断是基于LLM的推理，不是基于数据的统计
- 用户可能觉得"AI怎么知道我适合什么？"
- 没有大量用户数据验证准确性

**应对方案**：
1. **透明化推理过程**：不只给结果，展示"为什么推荐这个方向"的推理链
2. **置信度标注**：每个推荐标注"匹配度XX%"，让用户自己判断
3. **可验证的案例**：每个推荐附一个真实案例，用户可以自己去查证
4. **小样本验证**：比赛前找10-20个真实用户测试，收集反馈

**诚实判断**：准确性很难"验证"，但可以通过透明化和可验证性建立信任。

#### 风险2：AI幻觉可能导致什么后果？如何兜底？

**风险等级**：高

**最坏情况**：
- 推荐一个完全不适合的方向（如推荐内向的人做销售）
- 用户真的去尝试，浪费时间甚至错过其他机会
- 用户对产品失去信任

**兜底机制**：

1. **预置方向库**（见4.3节）
   - AI只从预定义的方向中选择，不自己生成新方向
   - 过滤掉匹配度低于60%的推荐

2. **反向校验**
   - 如果AI推荐的方向需要的能力与用户明确表示"不擅长"的能力冲突，自动降权

3. **免责声明**
   - 明确告知用户：AI推荐仅供参考，最终决策需要自己验证
   - "这只是帮你打开思路，不是给你的人生答案"

4. **人工审核兜底**（可选）
   - 你作为产品负责人，定期检查AI推荐质量
   - 如果发现明显错误，及时调整Prompt

   - 用户填问卷，你手动发报告
   - 用于小样本验证
   - 实现难度：0天

**建议**：先做降级2（Coze Bot嵌入），确保5.31前有一个"能跑的AI Demo"。然后再迭代到降级1或完整版。

### 5.3 竞赛层面的风险

#### 风险1：如果评委觉得"这不就是套了AI壳子的职业测试"怎么办？

**风险等级**：高

**应对方案**：

1. **强调认知层创新**
   - 不是"AI+职业测试"，是"AI+认知框架"
   - 你的道法术器、五种货币、六条路径是原创框架，不是通用测评
   - AI是帮你用这个框架做推理，不是给你贴标签

2. **展示AI推理过程**
   - 不只给结果，展示"AI是怎么想到这个推荐的"
   - 让评委看到真实的推理链，而非黑箱

3. **展示用户调研数据**
   - 你有真实的用户调研报告（85+条UGC数据）
   - 有心理学理论支撑
   - 不是拍脑袋做的

4. **差异化定位**
   - 明确说出"我们不做MBTI/霍兰德那样的标签化测评"
   - 我们做的是"帮你建立自己的认知框架"

#### 风险2：如果其他选手也做了类似的东西？

**风险等级**：中等

**应对方案**：

1. **你的差异化**
   - 认知层蓝海：市场上没有产品做"引导式对话+个人画像"
   - 心理学背景：你的框架有理论支撑，不是瞎编的
   - 真实案例库：18个可溯源的案例，不是AI瞎编的

2. **即使功能相似，你的叙事不同**
   - 别人：AI职业测评 → 标签化、一次性
   - 你：AI认知引导 → 过程化、持续性

3. **如果真的撞车**
   - 强调你的"一人公司"模式——一个人能做出来，说明产品轻量、可复制
   - 强调你的"反共识"视角——不是跟风，是独立思考的结果

#### 风险3：这个项目在"数字赋能"分类里的竞争烈度预估

**预估**：中等偏高

**理由**：
- "数字赋能"是热门赛道，很多团队会做AI+教育/就业
- 但大部分团队会做"AI+工具"（简历生成、面试模拟），而非"AI+认知"
- 你的"认知层"定位是差异化，但评委可能不理解

**建议**：
- 在路演中快速建立"认知层"的概念
- 用对比展示：AI简历工具 vs AI认知引导
- 让评委在30秒内理解"为什么这个更有价值"

# 附录：诚实的价值判断

## AI在这个场景下真正有用的地方

1. **信息结构化**：把碎片化的就业信息组织成可筛选的选项
2. **个性化推理**：基于用户具体条件做匹配，而非通用建议
3. **苏格拉底追问**：帮用户澄清模糊的价值观（比用户自己想更系统）
4. **行动具体化**：把"找实习"拆成"明天早上做这1件事"

## AI在这个场景下没那么有用的地方

1. **给答案**：用户需要的不是"你适合做什么"的答案，而是"怎么想清楚"的工具
2. **替代人的判断**：价值观排序、能量识别、风险偏好——这些必须用户自己做
3. **保证准确**：AI推理是概率性的，不是确定性的——必须告诉用户这一点

## 最诚实的结论

AI在这个项目中的价值不是"万能的"，而是在**三个环节有真实价值**：
1. 信息处理（选项分布、案例整理）
2. 推理辅助（概率校准、路径规划）
3. 具体化（行动拆解、报告生成）

在**价值观判断、能力感知、能量识别**等主观环节，AI只能做"追问者"而非"回答者"。

**产品定位应该是**："AI帮你想清楚，而不是替你想"。

这是你的认知框架的核心——"道法术器"中，AI是"器"，不是"道"。用户必须自己建立自己的"道"。
